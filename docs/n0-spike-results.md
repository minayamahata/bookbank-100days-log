## 2. スコア下限（scoreFloor）調整のトレードオフ実例

**実例: 「容疑者Xの献身 ↔ 火車」エッジ（score 0.17）**

- 内訳: 共有キーワード「ミステリー」0.124（0.15 × idf 0.83）＋ 同月メモ 0.05 ＝ **0.174**
- このエッジは雑音辞書の修正を生き残った**正当なエッジ**（両冊のメモが「ミステリー」に言及し、同じ月に登録・月メモあり）。「意外なつながり」というこの機能の価値そのもの
- **教訓**: 下限を 0.15 → 0.2 に上げると、このような「単語1語＋月メモ」の正当な弱いエッジが一律に消える。雑音エッジ（同スコア帯に現れる）を消したい場合は、**まずストップワード辞書の増強で対処し、下限の引き上げは最後の手段とする**——今回、雑音3エッジは辞書追記だけで消え、正当な 0.17 エッジは残った。これは第7章判断表の「第一候補: 辞書 / 第二候補: 下限」の順序を実証する結果
- 実本棚評価（手順7）で下限を動かす場合は、この帯域（0.15〜0.25）の正当エッジがどれだけ巻き添えになるかを必ず数えること

- データ: 実機エクスポート（`tools/n0-spike/data/` 配下・**コミット禁止**）。425冊・書籍メモ12件・月別メモ1件・著者あり399冊・**seriesNameあり0冊**（APIからseriesNameがほぼ取れていない＝S-Aは実本棚では機能せず、シリーズ判定は実質S-Bのみ）

### 実本棚で検出し修正した辞書漏れ（手順4の延長として対処）

1. **巻数パターン**: 「新世界より(中)」「いつかティファニーで朝食を 14(完)」の `(上|中|下)`・`(完)`・`(全N巻)` 形式が未対応で基底タイトルに残留 → パターン追加。修正で新世界より3冊・空飛ぶタイヤ2冊等のクラスタが正しく成立
2. **ストップワード**: 頻度表上位に機能語の断片（「ない」「する」「できる」「って」等20語）が浮上 → 日本語辞書に追記。単語のみエッジが5.4%→2.9%に減少。テストデータ回帰（誤結合0%・取りこぼし25%）も維持

### 実本棚で確認された構造的な取りこぼし（手順7の議題として記録・未対処）

- **シリーズ名がタイトル末尾に付く形式**: 「月の影 影の海（上） 十二国記」等、十二国記16冊は共通**接尾辞**でシリーズを示すため、接頭辞ベースのS-Bでは全滅する。実本棚の取りこぼし率を大きく押し上げる見込みの最大要因
- **巻数がタイトル中間に入る形式**: 「ゴーストハント1　旧校舎怪談」（番号＋サブタイトル）は末尾アンカーのパターンで除去できず、接頭辞比率も60%に届かない（7冊）
- **著者表記の違いによる分断**: 「おひとりさまホテル」がマキヒロチ（原作）とまろ（作画）で2クラスタに分かれた。著者一致必須の設計上の想定内（計画メモ4.2節の測定外領域に近い分断の実例）
- **同一タイトルの重複登録**: 「クスノキの番人」2冊はシリーズではなく同じ本の二重登録。ラベル付け時の扱いに注意
- 韓国語断片「싶어」（〜したい）が頻度表に浮上。基準5（語抽出品質）の評価時に確認する

## 3.5 楽天APIの seriesName 活用案の検証（オフライン・不採用・2026-07-11）

### 動機
実本棚で seriesName が0冊だったため、「楽天ブックス書籍検索API（`endpoint=book`）が返す `seriesName` を取得してシリーズ判定S-Aを復活させれば、S-Bが構造的に拾えない末尾シリーズ名型（十二国記16冊等）を救えるのでは」という案が出た。アプリ配線は既に存在（`RakutenBook.seriesName` → `UserBook.seriesName` を `toUserBook` がマッピング済み。空だったのは検索が総合検索API（`endpoint=total`）を使い seriesName を返さないため）。着手前にオフラインで仮説検証した。

### 検証結果（**不採用**）
楽天プロキシの `endpoint=book` を実本棚のISBNで叩いた結果、`seriesName` が返す値は**作品シリーズ名ではなくレーベル名（文庫・コミックスの種別）**だった。

| 本 | 楽天 seriesName |
|----|----------------|
| 魔性の子 十二国記 | 新潮文庫　新潮文庫 |
| ハイキュー!! 1 | ジャンプコミックス |
| 大奥 1 | ジェッツコミックス |
| ゴーストハント1 | 角川文庫 |
| 容疑者Xの献身 | 文春文庫 |
| 白夜行 | 集英社文庫 |

無作為30冊の集計: **レーベル名25冊・空5冊・実シリーズ名0冊**。

### 判断根拠
- この値をS-A（seriesName一致・エッジ重み1.0）に流すと、「ジャンプコミックス」に属する全作品（ハイキュー＋鬼滅＋…）が**巨大な1つの誤シリーズに融合**する。誤結合は最強表示で最も目立つため、基準2（誤結合率2%以下）を破壊する
- 低重みの一般エッジとしても、「同じ文庫レーベル＝関連」は本棚上の意味を持たず、毛玉化（基準4）を悪化させるだけで価値がない
- 十二国記型の取りこぼしに効く、という当初の期待は成立しない（欲しい「十二国記」という語はレーベル名ではなくタイトル末尾にある）

### 含意（手順7の議題）
末尾シリーズ名型（十二国記・ゴーストハント）の救済は、外部データではなく**アルゴリズム内で完結させる必要がある**。候補は「同一著者内で共通する末尾トークンによる接尾辞クラスタリングのS-Bへの追加」。誤結合リスク（同じ著者の別シリーズを末尾語の偶然一致で繋がないか）とのトレードオフになるため、手順7で取りこぼし率の実測値を見てから是非を判断する。なおこれらは現状でも著者エッジ（0.6）では繋がっているため、無対応でも体験上の害は限定的（設計書2.2節）。

### 5.1 基準1（誤リンク率）の上界論証（目視サンプリングは非実施）

手順6の目視サンプリング（3値評価）は今回実施せず、エッジ構成の全数的性質から誤リンク率の**上界**を示してクローズする。全エッジは3種の軸から成る:

1. **シリーズエッジ（重み1.0）**: 4.1節の全数検査で**誤結合0%**。誤リンクを1本も含まない
2. **著者エッジ（重み0.6）**: 「同一著者の本を結ぶ」という生成条件そのものが正しさの定義であり、**構造的に誤リンクが起こり得ない**（著者名の同定さえ正しければ、結ぶこと自体が常に妥当）
3. **単語のみエッジ**: 全エッジの**2.9%**（基準4実測）

したがって、**仮に単語のみエッジがすべて誤リンクだったという最悪ケースでも、全体の誤リンク率は最大2.9% ≤ 5%** で基準1を満たす。かつ実際には単語エッジの多くは妥当（2章の「容疑者Xの献身↔火車」のような意味のある弱いエッジを含む）ため、真の誤リンク率は2.9%を大きく下回る。「不快レベル≤1%」も、私的な語による誤結合を生む雑音をストップワード辞書の増強で沈静させた（1章・3章）ことから、この上界の内側に収まると判断する。

### 5.2 手順6の目視サンプリングをR5解凍時に回す理由

- 現時点のエッジ構成は**タグエッジ（T2）を含まない**。ロードマップ2026-07-19改訂でメモタグ（R4.5）がノード（R5）より先に入るため、**R5β時点の実エッジ構成はタグエッジ込み**になる。現構成でサンプリングしてもβ時点を代表しない
- よって手順6の3値目視サンプリング（納得/中立/誤リンク）は、**R5解凍時にタグエッジ込みの実エッジ構成で実施する**タスクとして申し送る（8.3節）

## 7. R5解凍時への申し送り（既知の穴・改善候補）

1. **末尾シリーズ名型の救済**（最大の取りこぼし要因・105組）: 同一著者内の共通末尾トークンによる接尾辞クラスタリングをS-Bに追加する案（3.5節の含意）。誤結合リスクとのトレードオフを、この確定パラメータを基準線として評価すること
